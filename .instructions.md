# CloudQA Outbound Campaign - Complete Context & Workflow

## Campaign Overview

**Project:** Full-funnel LinkedIn-to-email lead generation and outreach pipeline for VP Engineering, CTOs, and Head of Engineering roles in North American companies (50-500 employees).

**Primary Goal:** Build qualified lead list + optimize for reply rates (not email opens) through 4-touch sequence.

**Overall Flow:** LinkedIn scrape → Connect → Enrich profiles → Find emails → Verify emails → Warmup account → Send 4-touch sequence

## Pipeline Scripts & Workflow

### 1. LinkedIn Connection (linkedin_connect.js)

**Purpose:** Scrape LinkedIn page for engineering leaders and send connection requests.

**Configuration:**
- Target Page: LinkedIn search results (specific page URL passed as argument)
- Action: Scrapes visible profiles, extracts name/title/company, sends connection requests with personalized note
- Timeout: 30-45 seconds per action to avoid bot detection
- Rate Limiting: Built-in delays between actions (3-5 second pauses)

**Execution:**
```bash
node linkedin_connect.js <linkedin_search_url>
```

**Output:**
- Console logs: Name, title, company, connection request status
- Side Effect: Sent connection requests (appears in LinkedIn outbox)
- Lead Data: Stored temporarily in memory; **requires manual transfer to leads list**

**Use Case:**
- Initial lead generation from LinkedIn search results
- Build raw prospect list before enrichment
- Typical Yield: ~10-20 profiles per page depending on filter

### 2. Profile Enrichment (profile_enricher.js)

**Purpose:** Enrich LinkedIn lead profiles by visiting LinkedIn profile URL and extracting company domain/website.

**Input:** CSV with LinkedIn profile URLs
- Required Field: `ProfileUrl` (full LinkedIn URL like `https://www.linkedin.com/in/name/`)
- Expected Fields: Name, Title, Company

**Logic:**
- Visits each LinkedIn profile URL
- Extracts visible company info and website
- Attempts to find company domain (e.g., cloudqa.io from company website)
- Updates CSV with enriched `Domain` field

**Execution:**
```bash
node profile_enricher.js
```

**Configuration:**
- Input File: `data/leads.csv` (modify script if different filename)
- Output: Same CSV with additional `Domain` column populated

**Output Columns Added:**
- `Domain`: Company domain (e.g., cloudqa.io)

**Typical Results:**
- Success Rate: ~60-80% (some profiles don't have company website listed)
- Time Per Profile: ~5-10 seconds
- Total Time for 100 profiles: ~10-15 minutes

### 3. Email Finding (email_finder.js)

**Purpose:** Find business email addresses for leads using Hunter.io API and other services.

**Input CSV Fields Required:**
- `FirstName`, `LastName`, `Domain` (company domain from enrichment step)
- `Email` (optional, for skipping if already found)

**API Services Used:**
1. **Hunter.io** (primary)
   - Finds verified company emails
   - Returns confidence level (high, medium, low)
   - API Key: Set as environment variable or hardcoded in script

2. **Fallback Logic:**
   - Catch-all detection (if Hunter succeeds but email not found)
   - Pattern guessing (firstname.lastname@domain, etc.)

**Execution:**
```bash
node email_finder.js
```

**Configuration:**
- Requires Hunter.io API key
- Rate Limiting: Built-in delays to respect API quotas
- Input: `data/leads_with_domains.csv`
- Output: `data/leads_with_emails.csv` (new file or updated)

**Output Columns Added:**
- `Email`: Found business email
- `Confidence`: "hunter_verified", "catch_all_guess", "pattern_guess", "not_found"

**Typical Results:**
- Hunter Verified: ~40-60% of leads
- Catch-all Guesses: ~10-15%
- Pattern Guesses: ~5-10%
- Not Found: ~20-30%

## Execution Workflow (Step-by-Step)

### Phase 1: Lead Generation (One-Time)
1. **LinkedIn scraping** → `node linkedin_connect.js <url>` → Get raw profiles
2. **Manual transfer** → Add profiles to `leads.csv`
3. **Profile enrichment** → `node profile_enricher.js` → Extract company domains
4. **Email finding** → `node email_finder.js` → Find business emails
5. **Email verification** → Reoon API verification (integrated or separate)
6. **Output:** `leads_with_emails.csv` with 675 leads (266 reoon_verified)

### Phase 2: Account Warmup (Pre-Campaign, ~5 days)
1. For each sender account:
   - `node zoho_warmup.js <sender_email>`
   - Send 5-10 emails daily to internal addresses
   - Build sender reputation
   - Typical Duration: 3-5 days per account

### Phase 3: Campaign Execution (Ongoing, ~14+ days per lead)
1. **Day 0 - Initial Send (Touch 1):**
   - `node zoho_send_emails.js akulkarni@cloudqa.us` → 5 emails
   - `node zoho_send_emails.js paige@cloudqa.us` → 5 emails
   - (Continue with other senders as needed)
   - Monitor for replies

2. **Day 3 - Follow-up (Touch 2):**
   - Script auto-triggers for leads that received Touch 1 3+ days ago
   - `node zoho_send_emails.js akulkarni@cloudqa.us`
   - Script filters to only send Touch 2 to eligible leads

3. **Day 7 - Second Follow-up (Touch 3):**
   - Auto-triggered for Touch 1 sent 7+ days ago
   - Same execution pattern

4. **Day 14 - Final Touch (Touch 4):**
   - Last outreach message
   - Same execution pattern

### Phase 4: Monitoring & Optimization
- Track reply rates daily
- Identify which A/B variants perform best (subject lines, body copy)
- Monitor bounce rate (should be <2% with reoon_verified only)
- Record reply content for sales team follow-up

## Debugging & Troubleshooting

### Email Copy Issues
- **Problem:** Emails show `${company}` or unreplaced variables
  - **Solution:** Check `buildEmail()` function in script; ensure template substitution is working
- **Problem:** Compliance language appearing in emails
  - **Solution:** Email copy not updated from old template; regenerate from this instructions file
- **Problem:** Placeholder brackets like `[industry]` in sent emails
  - **Solution:** Remove all `[bracketed]` placeholder logic; only use dynamic variables

### Sequence Issues
- **Problem:** Leads not advancing to Touch 2/3/4
  - **Solution:** Verify `sent_emails_log.json` is updating after each send; check timing windows (T2 = T1+3 days minimum)
- **Problem:** Script skipping leads or sending duplicates
  - **Solution:** Check `nextTouch()` logic; ensure touch history is being logged properly

### Deliverability Issues
- **Problem:** High bounce rate (>5%)
  - **Solution:** Verify using only reoon_verified leads; check if Reoon verification is actually running
- **Problem:** Emails landing in spam
  - **Solution:** Run zoho_warmup.js for longer period; check SPF/DKIM setup; rotate sender accounts more frequently

### Zoho Connection Issues
- **Problem:** Zoho login fails or UI automation times out
  - **Solution:** Verify sender account credentials; check if Zoho UI selectors have changed; ensure Playwright headless mode is working
- **Problem:** Compose window not opening
  - **Solution:** Wait times may be too short; increase timeouts in script

## Important Notes

4. **Don't manually edit log files** - Let script handle all state tracking; manual edits cause sync issues
5. **CloudQA positioning is non-negotiable** - All copy must reflect actual product (AI-powered, codeless, self-healing) not inferred pain points
6. **Reoon verification is critical** - Never send to unverified emails; bounce rate and spam complaints will destroy sender reputation
